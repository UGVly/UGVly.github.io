这篇论文的正文里，真正显式写出来的公式并不多，核心只有三块：

  1. 检索分数；
  2. 变换网络损失；
  3. 合成网络感知损失。

但如果想真正把方法吃透，必须把论文里隐含的对象、算子和训练流程一并补全。下面我按“问题定义 -> 检索 -> 对齐 -> 排序与合成 -> 最终生成 -> 训练目标”的顺序，把整条链完整写一遍。

1. 这篇论文到底在做什么

给定一个语义布局

L∈{0,1}h×w×c,L \in \{0,1\}^{h\times w\times c},L∈{0,1}h×w×c,

其中 h×wh\times wh×w 是图像分辨率，ccc 是语义类别数，目标是合成一张真实感图像 I^∈Rh×w×3\hat I\in\mathbb R^{h\times w\times 3}I^∈Rh×w×3。

传统纯参数方法是直接学

I^=gθ(L),\hat I = g_\theta(L),I^=gθ​(L),

也就是让网络把“所有外观细节”都记在参数里。

本文的方法不是这样。它把生成拆成两部分：

  • 非参数部分：从训练集构造一个外部 memory bank，测试时根据输入布局从里面检索真实图像片段；
  • 参数部分：用神经网络把这些检索到的片段对齐、排序、合成，并进一步修补与润色。

因此整条推理链写成

L  ⟹  {Lj}j=1m  ⟹  {Pσ(j)}j=1m  ⟹  {P~σ(j)}j=1m  ⟹  C  ⟹  I^.L \;\Longrightarrow\; \{L_j\}_{j=1}^m \;\Longrightarrow\; \{P_{\sigma(j)}\}_{j=1}^m \;\Longrightarrow\; \{\tilde P_{\sigma(j)}\}_{j=1}^m \;\Longrightarrow\; C \;\Longrightarrow\; \hat I.L⟹{Lj​}j=1m​⟹{Pσ(j)​}j=1m​⟹{P~σ(j)​}j=1m​⟹C⟹I^.

这里：

  • {Lj}j=1m\{L_j\}_{j=1}^m{Lj​}j=1m​ 是输入布局中的连通语义区域；
  • Pσ(j)P_{\sigma(j)}Pσ(j)​ 是为第 jjj 个区域从 memory 中检索到的真实图像片段；
  • P~σ(j)\tilde P_{\sigma(j)}P~σ(j)​ 是经过空间变换网络对齐后的片段；
  • CCC 是把所有片段按遮挡顺序贴到一起得到的 canvas；
  • I^\hat II^ 是最终合成图像。

3. 论文方法的总公式

如果把整篇论文写成一个统一的复合映射，那么测试时的生成过程可以记为

I^=fθf ⁣(C(L), L),\hat I = f_{\theta_f}\!\big(C(L),\,L\big),I^=fθf​​(C(L),L),

其中 canvas C(L)C(L)C(L) 是由若干中间步骤得到的：

这里：

  • σ(j)\sigma(j)σ(j) 是检索索引；
  • TθTT_{\theta_T}TθT​​ 是 transformation network；
  • Tj\mathcal T_jTj​ 表示目标布局信息，至少包含 LLL 与目标局部 mask LjmaskL_j^{\mathrm{mask}}Ljmask​；
  • π\piπ 是 ordering network 给出的从后到前的绘制顺序；
  • fθff_{\theta_f}fθf​​ 是最终 synthesis network。

这套写法有一个非常重要的性质：

检索步骤 σ(j)\sigma(j)σ(j) 直接访问外部数据库 MMM，因此模型不是纯参数模型，而是 semi-parametric。

4. 外部 Memory 的检索公式

4.1 查询对象

对测试布局 LLL 的每个连通分量 LjL_jLj​，构造与训练片段同构的两个量：

Ljmask,Ljcont.L_j^{\mathrm{mask}},\qquad L_j^{\mathrm{cont}}.Ljmask​,Ljcont​.

其中：

  • LjmaskL_j^{\mathrm{mask}}Ljmask​ 是该连通区域自身的 one-hot mask；
  • LjcontL_j^{\mathrm{cont}}Ljcont​ 是其在放大 bounding box 内的语义上下文。

4.2 IoU 的精确定义

论文把相似性建立在 IoU 上。对两个同形状的二值张量 A,B∈{0,1}h×w×cA,B\in\{0,1\}^{h\times w\times c}A,B∈{0,1}h×w×c，IoU 定义为

IoU⁡(A,B)=∣A∩B∣∣A∪B∣=∑x,kAx,kBx,k∑x,kmax⁡(Ax,k,Bx,k).\operatorname{IoU}(A,B) = \frac{|A\cap B|}{|A\cup B|} = \frac{\sum_{x,k} A_{x,k}B_{x,k}} {\sum_{x,k}\max(A_{x,k},B_{x,k})}.IoU(A,B)=∣A∪B∣∣A∩B∣​=∑x,k​max(Ax,k​,Bx,k​)∑x,k​Ax,k​Bx,k​​.

由于 A,BA,BA,B 是二值张量，也可写成

IoU⁡(A,B)=⟨A,B⟩∥A∥1+∥B∥1−⟨A,B⟩.\operatorname{IoU}(A,B) = \frac{\langle A,B\rangle} {\|A\|_1+\|B\|_1-\langle A,B\rangle}.IoU(A,B)=∥A∥1​+∥B∥1​−⟨A,B⟩⟨A,B⟩​.

4.3 论文中的检索分数

对第 jjj 个测试片段，候选集合限制在同语义类：

Mj:={i:κ(i)=κ(j)}.\mathcal M_j:=\{i:\kappa(i)=\kappa(j)\}.Mj​:={i:κ(i)=κ(j)}.

然后定义分数

s(i,j)=IoU⁡(Pimask,Ljmask)+IoU⁡(Picont,Ljcont).s(i,j) = \operatorname{IoU}\big(P_i^{\mathrm{mask}},L_j^{\mathrm{mask}}\big) + \operatorname{IoU}\big(P_i^{\mathrm{cont}},L_j^{\mathrm{cont}}\big).s(i,j)=IoU(Pimask​,Ljmask​)+IoU(Picont​,Ljcont​).

最终检索结果是

σ(j)=arg⁡max⁡i∈Mjs(i,j).\sigma(j) = \arg\max_{i\in\mathcal M_j} s(i,j).σ(j)=argi∈Mj​max​s(i,j).

这就是论文在 Section 4.2 的核心公式。

4.4 这个公式为什么合理

这个分数由两部分组成：

  1. mask IoU
IoU⁡(Pimask,Ljmask)\operatorname{IoU}\big(P_i^{\mathrm{mask}},L_j^{\mathrm{mask}}\big)IoU(Pimask​,Ljmask​)

衡量候选片段形状和目标片段形状是否匹配。

  1. context IoU
IoU⁡(Picont,Ljcont)\operatorname{IoU}\big(P_i^{\mathrm{cont}},L_j^{\mathrm{cont}}\big)IoU(Picont​,Ljcont​)

衡量片段周围语义环境是否匹配。

于是，作者不是单纯找“形状最像”的车，而是找“形状像、而且出现在类似周边环境中的车”。

在数学上，这个最大化问题等价于最小化一个加权 Jaccard 距离：

σ(j)=arg⁡min⁡i∈Mj[(1−IoU⁡(Pimask,Ljmask))+(1−IoU⁡(Picont,Ljcont))].\sigma(j) = \arg\min_{i\in\mathcal M_j} \Big[ \big(1-\operatorname{IoU}(P_i^{\mathrm{mask}},L_j^{\mathrm{mask}})\big) + \big(1-\operatorname{IoU}(P_i^{\mathrm{cont}},L_j^{\mathrm{cont}})\big) \Big].σ(j)=argi∈Mj​min​[(1−IoU(Pimask​,Ljmask​))+(1−IoU(Picont​,Ljcont​))].

也就是说，它本质上是一个基于形状与上下文的 nearest-neighbor retrieval。

4.5 多样性版本

论文最后说，如果要生成多样结果，不必总取全局最优 arg⁡max⁡\arg\maxargmax，而可以在 top-kkk 候选里随机采样：

σk(j)∼Unif⁡(TopK⁡i∈Mjs(i,j)).\sigma_k(j)\sim \operatorname{Unif}\Big( \operatorname{TopK}_{i\in\mathcal M_j} s(i,j) \Big).σk​(j)∼Unif(TopKi∈Mj​​s(i,j)).

这样同一个布局 LLL 可以生成多种风格不同但语义兼容的图像。

5. 变换网络 TTT：从检索到对齐

检索出的真实片段虽然语义类与上下文都匹配，但其位置、尺度、旋转角度通常与当前输入布局不一致，因此必须做几何对齐。

5.1 网络输入输出

论文把 transformation network 记为

T(L,  Ljmask,  Pσ(j)color),T\big(L,\;L_j^{\mathrm{mask}},\;P_{\sigma(j)}^{\mathrm{color}}\big),T(L,Ljmask​,Pσ(j)color​),

输出为对齐后的片段

P~σ(j).\tilde P_{\sigma(j)}.P~σ(j)​.

更抽象地，我们记目标条件为

Tj:=(L,  Ljmask),\mathcal T_j := \big(L,\;L_j^{\mathrm{mask}}\big),Tj​:=(L,Ljmask​),

则

P~σ(j)=TθT(Tj,  Pσ(j)color).\tilde P_{\sigma(j)} = T_{\theta_T}\big(\mathcal T_j,\;P_{\sigma(j)}^{\mathrm{color}}\big).P~σ(j)​=TθT​​(Tj​,Pσ(j)color​).

5.2 仿射变换的标准写法

论文明确说这里使用的是 2D affine transformation。因此，网络本质上是预测一个仿射矩阵

Aj=[a11a12txa21a22ty].A_j = \begin{bmatrix} a_{11} & a_{12} & t_x\\ a_{21} & a_{22} & t_y \end{bmatrix}.Aj​=[a11​a21​​a12​a22​​tx​ty​​].

对目标坐标 (xt,yt)(x_t,y_t)(xt​,yt​)，其对应的源坐标 (xs,ys)(x_s,y_s)(xs​,ys​) 满足

[xsys]=Aj[xtyt1].\begin{bmatrix} x_s\\ y_s \end{bmatrix} = A_j \begin{bmatrix} x_t\\ y_t\\ 1 \end{bmatrix}.[xs​ys​​]=Aj​​xt​yt​1​​.

5.3 STN 的双线性采样公式

若把待变换图像记为 U=Pσ(j)colorU=P_{\sigma(j)}^{\mathrm{color}}U=Pσ(j)color​，输出图像记为 V=P~σ(j)V=\tilde P_{\sigma(j)}V=P~σ(j)​，则对通道 ccc 有

Vc(xt,yt)=∑m=1h∑n=1wUc(m,n) max⁡(0,1−∣xs−n∣) max⁡(0,1−∣ys−m∣).V_c(x_t,y_t) = \sum_{m=1}^h\sum_{n=1}^w U_c(m,n)\, \max(0,1-|x_s-n|)\, \max(0,1-|y_s-m|).Vc​(xt​,yt​)=m=1∑h​n=1∑w​Uc​(m,n)max(0,1−∣xs​−n∣)max(0,1−∣ys​−m∣).

  • 对仿射参数可微；
  • 对输入像素值也可微。

因此可以端到端地反向传播更新 θT\theta_TθT​。

5.4 训练样本怎么构造

如果直接拿一个正确片段 PicolorP_i^{\mathrm{color}}Picolor​ 让网络去对齐 PimaskP_i^{\mathrm{mask}}Pimask​，任务会过于平凡，因为它本来就是对齐的。

论文的处理方式是：对 PicolorP_i^{\mathrm{color}}Picolor​ 施加随机仿射扰动与裁剪，得到

P^icolor=Wrand(Picolor).\hat P_i^{\mathrm{color}} = W_{\mathrm{rand}}\big(P_i^{\mathrm{color}}\big).P^icolor​=Wrand​(Picolor​).

于是训练任务变成：

  • 输入：被扰乱过的片段 P^icolor\hat P_i^{\mathrm{color}}P^icolor​ 与目标布局条件；
  • 输出：恢复到原始位置与形状的片段。

5.5 论文里的变换损失

论文给出的损失是

LT(θT)=∑Pi∈M∥Picolor−TθT(Ti,P^icolor)∥1.L_T(\theta_T) = \sum_{P_i\in M} \left\| P_i^{\mathrm{color}} - T_{\theta_T}\big(\mathcal T_i,\hat P_i^{\mathrm{color}}\big) \right\|_1.LT​(θT​)=Pi​∈M∑​​Picolor​−TθT​​(Ti​,P^icolor​)​1​.

这里 Ti\mathcal T_iTi​ 表示与第 iii 个 memory 片段对应的目标布局信息。论文版面中在这一处的记号略有压缩，但上下文一致的理解就是：

给定目标 mask / layout，以及一个几何上被扰乱的片段，让网络把它对齐回正确位置。

5.6 为什么用颜色图像做 L1L_1L1​ 而不是只对 mask 做监督

如果只监督 mask，对齐会有大量不唯一解。比如一个车片段，单从 mask 很难分辨“正着放还是翻转后放”，而颜色纹理提供了更强的约束。

所以作者强调：损失定义在 color image 上，比定义在 mask 上更具体。

从优化角度看，L1L_1L1​ 损失的梯度是

∇θTLT=∑iJθT ⁣[TθT(Ti,P^icolor)]⊤sgn⁡ ⁣(TθT(Ti,P^icolor)−Picolor),\nabla_{\theta_T} L_T = \sum_i J_{\theta_T}\!\Big[T_{\theta_T}(\mathcal T_i,\hat P_i^{\mathrm{color}})\Big]^\top \operatorname{sgn}\!\Big( T_{\theta_T}(\mathcal T_i,\hat P_i^{\mathrm{color}}) - P_i^{\mathrm{color}} \Big),∇θT​​LT​=i∑​JθT​​[TθT​​(Ti​,P^icolor​)]⊤sgn(TθT​​(Ti​,P^icolor​)−Picolor​),

其中 JθTJ_{\theta_T}JθT​​ 是输出对参数的 Jacobian。因为 STN 的采样过程可微，所以这条链完整可导。

6. Ordering Network 与 Canvas 合成

6.1 为什么需要排序

若所有变换后的片段都互不重叠，那么直接贴到画布上就行。但现实情况是，它们会发生重叠：

supp⁡(P~σ(i))∩supp⁡(P~σ(j))≠∅.\operatorname{supp}(\tilde P_{\sigma(i)}) \cap \operatorname{supp}(\tilde P_{\sigma(j)}) \neq \varnothing.supp(P~σ(i)​)∩supp(P~σ(j)​)=∅.

一旦重叠，就必须决定谁在前，谁在后。

6.2 排序网络的输出

论文说 ordering network 的输出是一个 ccc 维 one-hot 向量，表示“哪一个语义类应该在前面”。把它写成概率向量：

pij=OθO(P~σ(i),P~σ(j),L)∈Δc−1.p_{ij} = O_{\theta_O}\big(\tilde P_{\sigma(i)},\tilde P_{\sigma(j)},L\big) \in\Delta^{c-1}.pij​=OθO​​(P~σ(i)​,P~σ(j)​,L)∈Δc−1.

设真实前景类标签为

yij∈{e1,…,ec},y_{ij}\in\{e_1,\dots,e_c\},yij​∈{e1​,…,ec​},

则 cross-entropy 损失为

LO(θO)=−∑(i,j)∈A∑r=1cyij,rlog⁡pij,r,L_O(\theta_O) = - \sum_{(i,j)\in\mathcal A} \sum_{r=1}^c y_{ij,r}\log p_{ij,r},LO​(θO​)=−(i,j)∈A∑​r=1∑c​yij,r​logpij,r​,

其中 A\mathcal AA 是训练时采样到的相邻片段对集合。

6.3 监督信号从哪来

论文的监督来自 segment 的相对深度顺序：

  • Cityscapes / NYU 这类数据集自带 depth 或 stereo 信息；
  • ADE20K 没有，就先用单目深度网络估计深度。

论文没有把“segment 级别深度标签如何从像素级深度聚合出来”写成公式；实现上只要能为相邻片段对给出一个可靠的 front/back 标号即可。

6.4 画布合成公式

设经过排序后，从后到前的绘制顺序为

π(1),π(2),…,π(m).\pi(1),\pi(2),\dots,\pi(m).π(1),π(2),…,π(m).

其中：

  • S={conv1_2,conv2_2,conv3_2,conv4_2,conv5_2}\mathcal S=\{\texttt{conv1\_2},\texttt{conv2\_2},\texttt{conv3\_2},\texttt{conv4\_2},\texttt{conv5\_2}\}S={conv1_2,conv2_2,conv3_2,conv4_2,conv5_2}；

8.2 为什么这是“感知损失”

如果直接做像素级 L1L_1L1​：

网络会倾向于平均化，常导致模糊。

而感知损失是在预训练视觉网络特征空间里逼近：

Φl(I^)≈Φl(I).\Phi_l(\hat I)\approx \Phi_l(I).Φl​(I^)≈Φl​(I).

低层特征约束颜色与边缘，高层特征约束纹理和结构，所以更适合照片合成任务。

由于 Φl\Phi_lΦl​ 冻结不训练，对 θf\theta_fθf​ 的梯度为

∇θfLf=∑(I,L)∑l∈Sλl Jθf ⁣[Φl(I^)]⊤sgn⁡ ⁣(Φl(I^)−Φl(I)),\nabla_{\theta_f}L_f = \sum_{(I,L)} \sum_{l\in\mathcal S} \lambda_l\, J_{\theta_f}\!\Big[\Phi_l(\hat I)\Big]^\top \operatorname{sgn}\!\Big(\Phi_l(\hat I)-\Phi_l(I)\Big),∇θf​​Lf​=(I,L)∑​l∈S∑​λl​Jθf​​[Φl​(I^)]⊤sgn(Φl​(I^)−Φl​(I)),

这说明优化时，误差会先通过固定的 VGG 特征网络反传回输出图像，再由合成网络继续反传回参数 θf\theta_fθf​。

9. 模拟训练画布 C′C'C′ 是怎么构造出来的

这是整篇论文非常关键、但很多人第一次读会忽略的地方。

作者没有在训练时直接把真实 memory 检索结果拿来当 canvas，而是从真实图像 III 出发，人为制造出与测试时相似的伪缺陷，构造模拟画布 C′C'C′。这样网络学到的是“修复与融合”的能力。

9.1 Step 1：Stenciling

对真实图像里的每个 segment PjP_jPj​，从别的图像里检索一个同类 segment，取其 mask 来裁切 PjP_jPj​：

Pjstencil=Pjcolor⊙Mr(j),P_j^{\mathrm{stencil}} = P_j^{\mathrm{color}}\odot M_{r(j)},Pjstencil​=Pjcolor​⊙Mr(j)​,

其中：

  • r(j)r(j)r(j) 是检索到的 donor segment；
  • Mr(j)∈{0,1}h×wM_{r(j)}\in\{0,1\}^{h\times w}Mr(j)​∈{0,1}h×w 是 donor 的二值 mask；
  • ⊙\odot⊙ 是逐像素乘法。

这一步人为制造“缺块”和“形状不一致”。

9.2 Step 2：Color transfer

论文在这里引用的是 Reinhard 等人的颜色迁移方法 [27]，正文没展开公式。若在 Lab 色彩空间中写，其标准形式是：

x~c=σr(j),cσj,c+ε(xc−μj,c)+μr(j),c,c∈{L,a,b}.\tilde x_c = \frac{\sigma_{r(j),c}}{\sigma_{j,c}+\varepsilon} \big(x_c-\mu_{j,c}\big) + \mu_{r(j),c}, \qquad c\in\{L,a,b\}.x~c​=σj,c​+εσr(j),c​​(xc​−μj,c​)+μr(j),c​,c∈{L,a,b}.

这里：

  • μj,c,σj,c\mu_{j,c},\sigma_{j,c}μj,c​,σj,c​ 是当前 segment PjP_jPj​ 在通道 ccc 上的均值与标准差；
  • μr(j),c,σr(j),c\mu_{r(j),c},\sigma_{r(j),c}μr(j),c​,σr(j),c​ 是 donor segment 的统计量。

这一步会故意制造 segment 之间的色调、亮度不一致。

9.3 Step 3：Boundary elision

定义 segment 边界为 ∂Ωj\partial\Omega_j∂Ωj​，到边界的距离为

论文说：对距离边界 0.05h0.05h0.05h 以内的 segment 内部像素，随机擦掉 80%80\%80%，并填成白色。

于是内部边界带可写为

若 Rj(x)∼Bernoulli⁡(0.8)R_j(x)\sim\operatorname{Bernoulli}(0.8)Rj​(x)∼Bernoulli(0.8)，则被擦掉的内部边界像素集合为

外部边界带定义为

9.4 最终模拟画布

把经过上述处理的各个训练片段按真实布局或对应顺序合成，就得到模拟测试时缺陷的画布

于是训练目标变成

(C′,L)↦I.(C',L)\mapsto I.(C′,L)↦I.

这个设计非常漂亮，因为它把任务从“从零生成整张图”转化为：

  • 以真实照片片段为原料；
  • 修补缺失区域；
  • 融合颜色与光照；
  • 消除边界穿帮。

这也是为什么它比当时很多纯参数方法更真实。

10. 三个训练目标分别是什么

这篇论文不是端到端联合训练，而是分模块训练：

10.1 变换网络

10.2 排序网络

因此它不是

这也解释了论文最后为什么把“not trained end-to-end”列为未来工作之一。

12. 结果怎么读

论文实验主要给出三类证据。

12.1 人工 A/B 偏好

平均来看：

  • SIMS 比 Pix2pix 更真实：94.1%
  • SIMS 比 CRN 更真实：86.1%

12.2 语义分割可解析性

把生成图再送进 PSPNet 做语义分割，比较其输出和原始布局的一致性。论文报告例如：

  • Cityscapes-coarse：SIMS 的 IoU 为 56.3%，明显高于 Pix2pix 的 30.1% 与 CRN 的 28.5%
  • ADE20K：SIMS 的 IoU 为 38.4%，高于 Pix2pix 的 16.1% 与 CRN 的 23.1%

这个实验很有意思，因为它同时检验：

  • 图像是否像真的；
  • 图像是否仍然忠实于输入布局。

12.3 频谱统计

作者比较了生成图像与真实图像的平均 power spectrum。SIMS 的频谱曲线与真实图像几乎重合，而 Pix2pix / CRN 则有明显尖刺。

13. 这篇论文为什么有效

我觉得可以从下面三点理解。

13.1 高频细节不是全靠网络“背下来”

纯参数模型需要在权重里记住所有纹理模式。SIMS 则在测试时直接从数据库里拿真实片段，因此：

  • 纹理更像真照片；
  • 高频统计更自然；
  • 细节生成压力从“凭空 hallucinate”变成“检索后融合”。

13.2 最终网络做的是“融合与修补”，不是“从零捏整张图”

这使得任务难度明显下降。网络主要负责：

  • 对齐后的局部 harmonization；
  • 缺块补全；
  • 边缘过渡；
  • 阴影和接触区域补充。

13.3 输入布局 LLL 与 canvas CCC 双重约束

若只看 canvas，网络可能会被 donor segment 带偏； 若只看 layout，又会退化成纯参数生成器。

两者同时输入后，网络既能利用真实 donor 的 appearance，又不会脱离目标语义布局。

14. 这篇论文的局限

论文自己也提到了，结合公式更容易看清问题在哪。

14.1 检索是离散的、不可微的

因为

σ(j)=arg⁡max⁡is(i,j)\sigma(j)=\arg\max_i s(i,j)σ(j)=argimax​s(i,j)

是离散选择，所以整条链不能从最终图像损失直接反传回 memory 检索阶段。

14.2 不是端到端训练

三套网络分别训练，导致：

  • 模块之间可能存在误差传播；
  • 下游网络只能被动适应上游检索/对齐的偏差。

14.3 计算慢

若对每个 query segment 直接在同类 memory 中做暴力检索，复杂度大致是

O(∣Mj∣⋅h⋅w⋅c).O\big(|\mathcal M_j|\cdot h\cdot w\cdot c\big).O(∣Mj​∣⋅h⋅w⋅c).

整张图有 mmm 个 segment，则整体检索量级大致是

O(∑j=1m∣Mj∣ h w c),O\Big( \sum_{j=1}^m |\mathcal M_j|\,h\,w\,c \Big),O(j=1∑m​∣Mj​∣hwc),

这也是论文明确承认其实现比纯参数方法慢得多的原因。

14.4 Pairwise ordering 不一定全局一致

排序网络本质上学的是局部 front/back 判断。若多个片段同时重叠，pairwise 预测可能在理论上产生循环偏好。论文没有详细展开这部分的全局一致化策略，这属于实现层面的未完全形式化之处。

15. 今天回头看，这篇论文最有价值的地方

虽然这是 2018 年的工作，但它的思想今天看依然很前沿：

  1. 它本质上是在做“视觉版 RAG”
    不是把所有知识压进参数，而是在推理时访问外部 memory。

  2. 它把“生成”改造成“检索 + 对齐 + 融合”
    这对高保真图像任务尤其自然。

  3. 它承认非参数库本身就是知识载体
    网络不用从头发明真实纹理，可以直接借用真实照片片段。
